# Contributing Guide

Thanks for your interest in contributing!

## Workflow

1. Fork the repository on [GitHub](https://github.com/)
2. Create a feature branch
3. Follow the [setup guide](setup.md) to get running locally
4. Make your changes
5. Run `npm test` to verify
6. Open a pull request

## Code Standards

- Use [ESLint](https://eslint.org/) for linting
- Use [Prettier](https://prettier.io/) for formatting
- Follow the [architecture patterns](../architecture/overview.md)

## Documentation

- Update relevant docs when changing behaviour
- API changes must update [endpoint docs](../api/endpoints.md)
- Schema changes must update [schema docs](../api/schemas.md)
